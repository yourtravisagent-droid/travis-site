Feature

Travis Security Hardening

An agent that reads your email, touches your calendar, knows your financial data, and can make purchases needs actual guardrails. Not "trust me." Real, structural defenses against prompt injection, data exfiltration, and identity spoofing. Layered on by default.

🧾
Behavioral rules
Exfiltration allowlist
Hard-coded in Travis's prompt: never transmit .env, Keychain items, sensitive data JSONs, memory directory, personal credentials. Permitted destinations are your own accounts only.
🪤
Canary tripwires
Planted in sensitive data
Random CANARY-… strings seeded in financial + loyalty data files. If a canary ever appears in an outbound message, a prompt injection has succeeded — and the monthly red-team catches it.
🛑
Bash sandbox
Structural exfil blocks
The Claude Code settings file carries a deny list blocking exfil-shaped shell invocations: cat-to-curl of secrets, Keychain-to-mail, memory-dir-to-net, canary-in-POST. Even if the behavioral layer slips, the tool layer refuses.
🚨
Kill switch + red-team
Known recovery, proven resilience
/halt from your DM boots every Travis job and kills in-flight browsers. Monthly red-team auto-sends canonical injection emails, then scans outbound logs. Pass or fail, you hear the result.
Your part
Trust, but verify. Travis runs the verify loop itself.
Installed by the standard setup; optional to disable.
